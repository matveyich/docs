Структура требований

1. Функциональное назначение
	- use cases
		- user stories
	- описание бизнес сущностей
	- алгоритмы
		- в т.ч. и влияние на представления
2. Представления
	- представления
		[базовое представление] - есть всегда
		[представление Х - алгоритм Х] - опционально возможно множество модификаций базового представления
		- область уведомлений
			- какие уведомления, при каких условиях из п.1 будут отображены
		- элементы управления
			- шаблон
			- основная кнопка
			- другие кнопки в основной области
			- наполнение кнопки действия
				- нюансы алгоритма наполнения кнопки
		- область тела представления
			- шаблон (1, 2 колонки), расположение блоков
				- описание каждого блока, если из шаблона не понятно
				- список/порядок выводимых полей детали/реестра
		- меню (наверно везде все-таки одинаковое)
3. Реализация
	- объекты системы
		- структуры данных
		- class/component diagrams
		- sequence diagrams
	- процессы
	- права доступа
4. Примеры
	- моделирование
